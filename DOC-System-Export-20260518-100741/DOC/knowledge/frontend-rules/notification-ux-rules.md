# Notification UX Rules

## Purpose
Consistent inbox/feed and channel preference experiences.

## Rules
- Separate transient toasts from durable notification inbox
- Notification center must support unread filtering and mark-all-read
- Provide per-channel preference controls (email, in-app, SMS, push)
- Deep-link every notification to canonical destination when possible

## Anti-patterns
- Toast-only notifications for critical actions
- Missing channel preference controls
