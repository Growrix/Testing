# Onboarding Rules

## Purpose
Define first-run activation UX with measurable completion outcomes.

## Rules
- Onboarding must declare activation goal and success criteria
- Use checklist model with per-step completion state
- Include skip/resume behavior without data loss
- Persist progress per user and tenant
- First-run empty states must route into onboarding checklist

## Required outputs
- onboarding checklist spec
- completion event names for analytics
- fallback path for returning users
