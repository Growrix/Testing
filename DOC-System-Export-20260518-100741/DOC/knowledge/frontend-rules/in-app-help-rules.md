# In-App Help Rules

## Purpose
Deliver contextual help without disrupting core workflows.

## Rules
- Choose help primitive by complexity: tooltip, popover, side panel, full guide
- Help copy must be role-aware and context-aware
- Every help surface must include dismiss and never-show-again controls where appropriate
- Escalation path to docs/support must be visible

## Anti-patterns
- Modal help interruptions during critical form submission
- Generic help content detached from current context
