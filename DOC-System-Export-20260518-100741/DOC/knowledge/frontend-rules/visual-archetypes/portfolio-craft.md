# Portfolio Craft

## Voice
- Confident, work-led, low ego.
- Project speaks first, narrative second.
- Selective: a few strong cases beats many weak ones.

## Density
- Spacious project showcase.
- Medium "About / Approach" pages.
- Dense process or capabilities sections only when required.

## Color Direction
- Restrained palette: high-contrast neutrals + one signature hue.
- Project case studies may locally override the accent to match the project brand (allowed and encouraged).

### Default token roles
- background:  near-white or near-black depending on mood
- surface:     pure white / graphite
- inset:       very subtle neutral
- primary:     signature studio hue
- accent:      contextual (per-case-study override allowed)
- destructive/success/warning/info: standard semantic palette

## Typography
- Display family: distinctive — serif or display sans, brand-defining.
- Body family: humanist sans.
- Display scale: 96 / 80 / 64 / 56 (oversized hero common).
- Heading scale: 40 / 32 / 24 / 20.
- Body scale: 18 / 16.
- Line-height: 1.0–1.05 display (oversized), 1.3 headings, 1.7 body.

## Spacing
- Section rhythm: 96–128px desktop / 64px tablet / 48px mobile.
- Generous case-study padding.

## Surface system
- Page base: flat neutral.
- Elevated: borderless cards; image dominance.
- Inset: pull-quote blocks, process steps.
- Overlay: full-bleed media lightbox.

## Radius
- 8–16; sharp-leaning for craft feel.

## Motion personality
- Cinematic, restrained.
- Macro 240–320ms (slower).
- Micro 140–200ms.
- Subtle parallax in case-study heroes.
- Cursor-follow accents allowed if subtle.
- Reduced-motion: instant transitions, no layout shift.

## Imagery direction
- High-quality project work (UI screens, photography, video reels).
- Aspect ratios standardized per case-study type.
- Avoid stock imagery entirely.

## Hero composition
- Oversized statement headline.
- Below: featured projects grid (2–4 visible).
- Optional: short capability strip.

## CTA hierarchy
- Primary: "View work" or "Start a project".
- Secondary: "About" or "Approach".

## Iconography
- Used sparingly; outline 1.5px.

## Case-study structure mandate
Every case study spec MUST include:
- Hero image with project name and role.
- Context (client, year, team).
- Problem.
- Approach.
- Solution media (screens, photos, video).
- Outcome (metrics or testimonial).
- Next-project navigator at the bottom.

## Forbidden in this archetype
- Templated stock visuals.
- Loud animations that distract from the work.
- Mixed visual systems across case studies (each can have its own accent, but layout primitives stay consistent).
