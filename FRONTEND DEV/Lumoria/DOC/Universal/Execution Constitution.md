# Execution Constitution

## Purpose

Keep project planning and execution deterministic, local, and documentation-backed.

## Core Rules

1. Plan before major implementation.
2. Keep canonical project docs inside `DOC/PROJECT PLAN/`.
3. Update the tracker only after the plan or implementation state truly changes.
4. Prefer reuse over net-new architecture when an existing codebase already exists.
5. Do not rely on chat-only memory for ongoing project work.
